# Refleksi Mid-Semester — Software Development

  ## Materi yang Paling Sulit

  ### 1. Resolusi Merge Conflict (Minggu 3)
  Awalnya sangat membingungkan melihat tanda `<<<<<<<`, `=======`,
  `>>>>>>>` di dalam file. Saya sempat takut salah hapus kode yang penting.

  **Cara Mengatasinya:**
  Saya berlatih membuat konflik secara sengaja di repositori percobaan
  sampai saya benar-benar paham bagian mana yang harus dihapus dan
  bagian mana yang dipertahankan.

  ### 2. Memahami Alur Pull Request (Minggu 4)
  Bingung dengan alur fork → branch → PR, terutama bedanya
  base dan compare branch.

  **Cara Mengatasinya:**
  Membaca ulang materi dan mencoba langsung di GitHub dengan
  repositori sendiri sebagai simulasi.

  ### 3. Struktur MVC (Minggu 6)
  Sulit membayangkan bagaimana tiga komponen saling terhubung
  tanpa melihat contoh nyata yang berjalan.

  **Cara Mengatasinya:**
  Membuat proyek sederhana dan menelusuri alur data dari Controller
  ke Model ke View secara step-by-step.

  ## Kesimpulan
  Dari Minggu 1 hingga 7, saya menyadari bahwa Git bukan hanya
  alat teknis, melainkan alat komunikasi antar developer. Setiap
  commit, branch, dan PR adalah bentuk dokumentasi kerja yang
  akan dibaca oleh orang lain — bahkan oleh diri sendiri di masa depan.
